# Born2beRoot Wiki - 包括的学習リソース

Born2beRoot プロジェクトの包括的なドキュメントへようこそ。このWikiは、42 Tokyoの学生がプロジェクトの理解、実装、評価防御のすべてをカバーし、さらにLinuxシステム管理の深い理解を得るために設計されている。

---

## このWikiの特徴

このWikiは単なるセットアップ手順書ではない。以下の観点から、Born2beRootの各要件を深く掘り下げている。

1. **「なぜ」の徹底解説**: 各セキュリティ設定がなぜ必要なのか、具体的な攻撃シナリオとセットで解説する
2. **実際の攻撃事例**: CVE番号や実際のセキュリティインシデントを引用し、設定の重要性を実感できるようにする
3. **Linuxカーネルの内部構造**: ブートプロセス、systemd、procfs、カーネルモジュールなど、表面的な操作の裏側にある仕組みを解説する
4. **ネットワークの詳細**: TCP/IPモデル、SSH暗号化の仕組み、iptablesのチェーン構造など、ネットワークの深い理解を促す
5. **ストレージの内部動作**: ファイルシステムのinode構造、dm-cryptの動作原理、LVMのextentマッピングなどを解説する
6. **Infrastructure as Code**: Terraformの概念を大幅に拡充し、AWS/GCP/Azureでの実装例、CI/CDとの統合、Ansibleとの役割分担を詳述する
7. **実践演習**: フォレンジック演習、トラブルシューティング問題、設計演習を通じて実践的なスキルを養う
8. **80問以上の理解度チェック**: シナリオベース問題、ネットワークトラブルシューティング問題、Terraformコード読解問題を含む

## 学習のマインドセット

### 「設定方法」ではなく「なぜ」を理解する

Born2beRootの評価では、設定が正しく動作することだけでなく、「なぜその設定が必要なのか」を説明する能力が問われる。例えば:

- **NG**: 「SSHのポートを4242に変更しました」
- **OK**: 「デフォルトのport 22はボットによる自動スキャンの標的となるため、4242に変更しました。ただし、これはsecurity through obscurityに過ぎず、本質的なセキュリティ向上にはnmapなどのポートスキャンで発見可能なため、公開鍵認証やFail2banとの併用が実務では推奨されます」

### 「動いた」で満足しない

設定が動くことは最低条件である。以下のレベルを目指すこと:

1. **Level 1 (不十分)**: コマンドをコピペして設定した。動いている。
2. **Level 2 (最低限)**: 各設定の目的を1文で説明できる。
3. **Level 3 (合格)**: 各設定がどのような攻撃を防ぐのか、具体的なシナリオで説明できる。
4. **Level 4 (優秀)**: 設定の限界と、実務での改善策を提案できる。
5. **Level 5 (卓越)**: Linux カーネルレベルでの動作原理を説明でき、IaCでの自動化方法も理解している。

このWikiはLevel 5を目指すための教材である。

- ターミナル（コマンドライン）の基本操作ができること
- テキストエディタ（nano, vim）で文字を入力・保存できること
- 英語のドキュメントを辞書を使いながら読む意欲があること

## 注意事項

- ドキュメントは日本語で記述されているが、技術用語は英語のまま記載している（初出時に日本語訳を併記）
- 実際の VM セットアップ手順（スクリーンショット付き）は含まれていない。概念的な説明とドキュメントのみである
- Terraform による IaC アプローチは概念的・教育的な説明であり、そのまま実行可能なコードではない部分がある
- セキュリティに関する情報は教育目的であり、不正アクセスに使用してはならない
- 記載されている攻撃手法やCVE情報は、防御を理解するための教育目的でのみ言及している

## 各ファイルの概要

### 01-background.md (背景知識)
プロジェクトに必要な全ての基礎概念を網羅的に解説する。仮想化技術、オペレーティングシステム、LVM、ディスク暗号化、ブートプロセス、ファイルシステム階層、ファイアウォール、SSH、sudo、AppArmor/SELinux、cron、パスワードポリシーに加え、Linuxカーネルの内部構造（プロセス管理、メモリ管理、LSM）、TCP/IPの詳細モデル、ストレージの内部構造（inode、ext4、dm-crypt動作原理）までを深掘りする。

### 02-prerequisites.md (前提知識)
Born2beRootに取り組む前に確認すべき知識を整理する。基本的なLinuxコマンド操作に加え、TCP/IPの4層モデルの詳細、ポートの概念、NATの仕組み、プロセスのライフサイクル（fork, exec, wait, signal）、systemdのユニットファイル構造、ジャーナルログの詳細な使い方までをカバーする。

### 03-requirements.md (要件詳細)
subject の全要件をチェックリスト形式で整理し、各要件の背景（なぜこの要件があるのか）を具体的な攻撃シナリオとセットで解説する。確認コマンドだけでなく、各設定が防ぐ攻撃の種類と、設定が不十分な場合のリスクを詳述する。

### 04-evaluation.md (評価防衛)
評価者が聞く可能性のある50問以上の質問リストと、各質問に対する模範回答（短い版と詳細版の両方）を提供する。デモンストレーションの具体的手順、よくある間違い回答とその修正、評価中のトラブル対応策も含む。

### 05-solution-terraform.md (Terraform詳解)
Infrastructure as Codeの観点からBorn2beRootを再解釈する。Terraformの基本概念に加え、ステートファイルの仕組み、モジュールの作成と利用、count/for_eachによるループ、データソース、リモートステート（S3 + DynamoDB）、CI/CDとの統合（Terraform Cloud, GitHub Actions）を詳解する。Born2beRootの各要件をAWS/GCP/Azureで実装する場合の具体的コードと、Ansibleとの役割分担の詳細も含む。

### 06-design.md (設計原則)
Born2beRootの各要件の背後にあるセキュリティ設計原則を深掘りする。DACとMAC、Linux Security Modules (LSM) の仕組み、実際のCVEとセキュリティインシデント事例、暗号化アルゴリズム（AES, SHA-256）の概要、パスワードのエントロピー計算、ブルートフォース攻撃の数学的解析を含む。

### 07-study-guide.md (学習ガイド)
効率的な学習方法と各トピックの要点に加え、実践的な演習を大幅に追加する。「サーバーがハッキングされた原因を調査せよ」型のフォレンジック演習、LVMのリサイズ演習、UFWルールの設計演習、設定ファイルのdiffレビュー演習を含む。

### 08-learning-goals.md (学習目標)
プロジェクトを通じて習得すべき知識とスキルのロードマップを提供する。各スキルの実務での応用先、キャリアパスとの接続、次に学ぶべき技術の推奨を含む。

### 09-comprehension-check.md (理解度チェック)
80問以上のQ&Aを提供する。基礎知識の確認問題に加え、シナリオベースの問題（「朝出社したらサーバーが応答しない」）、ネットワークトラブルシューティング問題、LVM操作問題、セキュリティ監査問題（「このsshd_configの問題点を指摘せよ」）、Terraformのコード読解問題を含む。
